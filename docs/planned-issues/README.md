# Planned Issues

These five files are fully-scoped issue bodies for known MVP gaps —
features the codebase currently has as placeholders with a `TODO`
comment pointing back here.

**These are not GitHub Issues yet.** They're markdown files so they can
ship with the repo and be reviewed in a PR like any other content.
Before starting work on any of them:

1. Copy the file's content into a new GitHub Issue.
2. Use the `title` and `labels` from the YAML frontmatter at the top of
   the file.
3. Delete the file from this folder in the same PR that closes the
   resulting issue (keeps this folder representing only *not-yet-filed*
   work).

| File | Issue title |
|---|---|
| `001-mobile-navigation.md` | [MVP] Mobile navigation menu |
| `002-booking-form.md` | [MVP] Booking form — date, service type, string/tension fields |
| `003-contact-form.md` | [MVP] Contact form with validation |
| `004-faq-content.md` | [MVP] FAQ page content |
| `005-pricing-content.md` | [MVP] Pricing page content |

If you think of another MVP gap while working on one of these, don't
fold it into the same issue — add a new file here (or a new GitHub
Issue directly, if you're ready to scope it fully) instead.
